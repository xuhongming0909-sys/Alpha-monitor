# 清理清单

本文档用于锁定“归档什么、保留什么、最终删除什么”，避免清理阶段误删仍在使用的文件。

## 1. 立即保留

这些内容会继续参与新项目运行，只是会被改名或迁移位置：

- `config.yaml`
- `package.json`
- `package-lock.json`
- `requirements.txt`
- `启动服务.js`
- `数据调度.py`
- `页面入口.html`
- `数据库路径.py`
- `数据抓取/`
- `策略中心/`
- `页面展示/`
- `消息推送/`
- `共享基础/`
- `命令工具/`
- `运行数据/`
- `运行日志/`
- `重构文档/`

## 2. 第一批归档

这些内容明显属于旧入口、运行产物或临时文件，应先移入 `归档/旧项目_20260320/`：

- `ah_output.json`
- `ah_output_clean.json`
- `bonds_output.json`
- `.tmp-runtime-冒烟检查.json`
- `tmp-node-out.log`
- `tmp-node-err.log`
- 根目录旧 React / Expo 残留文件的 Git 历史痕迹说明

## 3. 第二批归档

这些内容在中文入口稳定前先保留，稳定后归档：

- `启动服务.js`
- `数据调度.py`
- `页面入口.html`
- `数据库路径.py`
- `共享基础/旧版兼容/`
- 所有“兼容旧脚本入口”脚本说明和桥接逻辑

## 4. 最终删除目标

这些内容在中文入口、中文路径、中文脚本全部验证通过后，应从运行链路中移除：

- `共享基础/旧版兼容/` 中全部可执行兼容代码
- `共享基础/路径/*` 中旧根目录回退逻辑
- `共享基础/配置/*` 中非密钥类环境变量回退逻辑
- `命令工具/` 中“仅用于兼容旧入口”的转发实现
- 文档中所有“迁移中”“兼容旧版”描述

## 5. 最终保留目标

最终仓库保留以下类型内容：

- 中文命名的职责目录
- 中文命名的业务插件目录
- `config.yaml` 配置合同
- 生态强制英文文件
- 中文说明文档
- 必要运行数据与日志目录
- 仅供追溯的归档目录
