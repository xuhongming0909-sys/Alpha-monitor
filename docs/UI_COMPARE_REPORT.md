# UI 数据展示对比报告

**日期**: 2026-04-30
**对比对象**: 旧版看板 (`/legacy`) vs React 新版 (`/`) 

---

## 1. 总体结论

新版 UI 在**模块覆盖度**上已基本对齐旧版（10 个 Tab），但在**单模块字段深度**、**交互功能**、**数据密度**上存在明显差距。用户感知"数据太少"的主要来源：概览页聚合丢失、展开详情缺失、分页限制、部分字段被省略。

---

## 2. 模块级对比

### 2.1 概览页（Overview）

| 维度 | 旧版 | 新版 | 差距 |
|------|------|------|------|
| 结构 | 顶部状态栏 + 各模块独立表格 | 核心数据矩阵 + 机会排序 + 打新摘要 | 新版丢失各模块独立表格 |
| 转债展示 | 完整 20 列表格（含分页） | 仅 TOP 8 机会卡片（策略/标的/指标/数值/风险/报价） | **严重缩水** |
| AH/AB 展示 | 完整溢价表格（含分页） | 仅 TOP 5 机会卡片 | **严重缩水** |
| LOF 展示 | 完整 LOF 表格（含分页） | 仅 TOP 5 机会卡片 | **严重缩水** |
| 事件套利 | 完整表格（含分页） | 仅 TOP 3 机会卡片 | **严重缩水** |
| 打新展示 | 完整申购表格（含分页） | 仅今日申购/中签/上市统计卡片 | **严重缩水** |
| 自定义监控 | 完整监控表格（含分页） | **概览页完全不展示** | **缺失** |
| 分红提醒 | 完整分红表格（含分页） | **概览页完全不展示** | **缺失** |

**问题根因**: 新版概览页用 "机会排序" 聚合了所有模块的 TOP N，但每个机会只展示 7 个字段（策略、标的、代码、指标、数值、约束/风险、报价），用户无法看到完整表格。

## 4. 为什么用户觉得"数据太少"

1. **概览页降级**: 旧版概览页展示各模块完整表格，新版概览页只展示聚合后的 TOP N 机会卡片（7 个字段 vs 20 个字段）
2. **展开行缺失**: 旧版点击任意行可展开查看 10+ 个额外字段，新版无此功能
3. **分页限制**: 新版固定只展示前 50 条，旧版支持分页浏览全部数据
4. **模块缺失**: 新版概览页不展示自定义监控和分红提醒模块

---

## 5. 修复建议

### 5.1 高优先级（立即修复）

1. **概览页恢复完整表格**: 在概览页展示各模块的完整表格（至少展示转债、AH、AB、LOF、事件套利）
2. **展开行功能**: 为所有表格添加点击展开详情功能
3. **分页功能**: 为所有表格添加分页

### 5.2 中优先级（本周修复）

4. **概览页补充缺失模块**: 添加自定义监控和分红提醒的概览卡片
5. **监控编辑功能**: 恢复自定义监控的添加/编辑/删除功能
6. **推送设置编辑**: 恢复推送配置的编辑功能

### 5.3 低优先级（后续优化）

7. **模块注释**: 恢复页面底部的数据来源/计算公式/策略说明
8. **列宽调整**: 支持用户调整表格列宽
