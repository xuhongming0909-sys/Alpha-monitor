# Alpha Monitor 需求手册

## 1. 仪表盘首页
- 功能清单：
  - 打开首页后，直接看到真实市场数据、股债打新、推送设置和 6 个功能页。
- 数据库和数据来源：
  - 页面只读取现有 `/api/*` 接口，不新增专用页面接口。
- 业务规则（核心）：
  - 页面不能再是空壳。
  - 标题下方不再放长说明。
  - 顶部改成一行状态文字，内联显示 `HKD/CNY`、`USD/CNY`、十年期国债收益率、更新时间。
  - `推送设置` 不再占用首页顶部右侧区域，统一下沉到整页末尾。
  - 页面整体继续使用黑红科技风，但布局必须更规整、更适合长表阅读。
- UI界面：
  - 最上方是标题和状态文字。
  - 中上部是“股债打新”长表。
  - 中部是 6 个功能页签和对应数据面板。
  - 页面最末尾是简洁版“推送设置”。
- 配置项说明：
  - 首页入口：`config.yaml > presentation.dashboard_entry`

## 2. 功能页名称
- 功能清单：
  - 首页主导航固定为 6 个功能页。
- 数据库和数据来源：
  - 继续使用原有模块对应 API。
- 业务规则（核心）：
  - 名称固定为：
    - `转债套利`
    - `AH溢价`
    - `AB溢价`
    - `监控套利`
    - `分红提醒`
    - `收购私有`
- UI界面：
  - 6 个名称作为首页主 tab。
- 配置项说明：
  - 本轮名称是固定页面文案，不单独放配置项。

## 3. 顶部状态行
- 功能清单：
  - 用一行文字显示汇率和十年期国债收益率。
- 数据库和数据来源：
  - 汇率：`/api/market/exchange-rate`
  - 十年期国债收益率：`/api/market/convertible-bond-arbitrage`
- 业务规则（核心）：
  - 必须显示 `HKD/CNY`、`USD/CNY`、十年期国债收益率、更新时间。
  - 不显示十年期国债日期。
  - 接口没有返回国债收益率时，要明确写“暂未返回”。
- UI界面：
  - 状态信息是一整行文字，不再使用大状态卡。
- 配置项说明：
  - 汇率刷新频率：`config.yaml > data_fetch.plugins.exchange_rate.refresh_interval_ms`

## 4. 股债打新
- 功能清单：
  - 展示今天需要关注的打新事项。
- 数据库和数据来源：
  - 新股：`/api/market/ipo`
  - 可转债：`/api/market/convertible-bonds`
- 业务规则（核心）：
  - 只显示“今天相关”的事项。
  - 改成横向长表，不再拆成三张大卡片。
  - 表里至少显示：当前阶段、类型、名称/代码、申购日、中签缴款日、上市日、申购上限、发行价或转股价。
  - 页面不再单独显示 `抽签日` 标签。
  - 页面展示的 `中签缴款日` 列本轮固定使用 `lotteryDate` 的日期值。
  - `今日中签缴款` 阶段判断也必须与页面显示口径一致，按 `lotteryDate = 今天` 归类，不能再按 `paymentDate` 单独判定。
  - 行要按 `今日申购`、`今日中签缴款`、`今日上市` 标出当前阶段。
  - 不同阶段使用不同底色。
- UI界面：
  - 一个横向长表，行内混排新股和转债。
  - 每行都保留 `新股` 或 `转债` 标签。
- 配置项说明：
  - 刷新频率：`config.yaml > data_fetch.plugins.subscription.refresh_interval_ms`

## 5. 推送设置
- 功能清单：
  - 在页面上查看和修改推送时间。
- 数据库和数据来源：
  - 读取和保存：
    - `GET /api/push/config`
    - `POST /api/push/config`
- 业务规则（核心）：
  - 名称固定为 `推送设置`。
  - 只保留 3 个时间框。
  - 前 2 个时间框给常规模块主推送。
  - 第 3 个时间框给 `收购私有` 专报推送。
  - 页面要显示当前推送状态，但不再写长解释。
  - 保存后必须按接口返回结果立即回写到页面，不能只提示成功却不刷新显示值。
  - 页面重载后必须继续读取运行时配置，不能退回旧默认值。
- UI界面：
  - 页面末尾的一行式编辑区，无需独立大边框。
  - 只保留标题、状态、3 个时间框、保存按钮和刷新按钮。
  - 整体视觉比当前版本更收敛，避免大卡片占用页面高度。
- 配置项说明：
  - 默认推送时间：`config.yaml > notification.default_full_push_time`
  - 默认模块开关：`config.yaml > notification.enabled_modules`

## 6. 转债套利
- 功能清单：
  - 看可转债套利机会和关键指标。
- 数据库和数据来源：
  - `/api/market/convertible-bond-arbitrage`
- 业务规则（核心）：
  - 主表必须新增序号列。
  - 主表必须支持点击表头排序。
  - 主表必须支持分页，每页固定 50 条。
  - 表里固定显示：
    - `序号`
    - `转债代码`
    - `转债名称`
    - `正股代码`
    - `正股名称`
    - `转债现价`
    - `转债涨跌幅`
    - `正股现价`
    - `正股涨跌幅`
    - `转股价`
    - `转股价值`
    - `转股溢价率`
    - `双低`
    - `理论溢价率`
    - `评级`
    - `剩余年限`
    - `剩余规模(亿)`
    - `成交额(亿)`
    - `到期税前收益率`
    - `上市日`
    - `转股起始日`
  - 转股价值 = 正股价格 * 100 / 转股价
  - 转债溢价率 = (转债现价 / 转股价值 - 1) * 100
  - 双低值 = 转债现价 + 转债溢价率
  - 转股套利空间 > 2% 才算候选机会
- UI界面：
  - 电脑端优先展示紧凑摘要和长表。
  - 顶部摘要区参考 `AH` 的分栏秩序，但整体密度更高、更节省空间。
  - 顶部摘要区不能再用大块卡片堆满首屏，主表要尽早进入视野。
  - 手机端保留同一套表格结构，通过横向滚动查看。
- 配置项说明：
  - 刷新频率：`config.yaml > data_fetch.plugins.convertible_bond.refresh_interval_ms`
  - 推送分类：`config.yaml > strategy.convertible_bond.push_categories`

## 7. AH溢价
- 功能清单：
  - 看 A 股和 H 股之间的溢价排行。
- 数据库和数据来源：
  - `/api/market/ah`
  - 历史样本来自 `runtime_data/shared/premium_history.db`
- 业务规则（核心）：
  - AH 溢价 = (H 股人民币价 / A 股价 - 1) * 100
  - H 股人民币价 = H 股价格 * 港币兑人民币汇率
  - 页面顶部固定显示前三和倒数前三。
  - 主表保留完整列表。
  - 主表必须新增序号列。
  - 主表必须支持点击表头排序。
  - 主表必须支持分页，每页固定 50 条。
  - 主表固定显示：
    - `序号`
    - `A股代码`
    - `A股名称`
    - `H股代码`
    - `H股名称`
    - `A股价`
    - `H股价`
    - `对手人民币价`
    - `价差`
    - `溢价率`
    - `近三年百分位`
    - `样本区间`
  - 样本区间用压缩格式显示，例如 `250520-260321`。
  - `historyCount` 仍然表示“近三年可用样本数”，但主表不单独显示这一列。
- UI界面：
  - 和 AB 页面使用完全相同的样式和交互。
  - 顶部是双摘要，下方是主表和样本说明。
- 配置项说明：
  - 刷新频率：`config.yaml > data_fetch.plugins.ah_premium.refresh_interval_ms`
  - 推送名额：`config.yaml > strategy.ah_premium.push_top_n`

## 9. 监控套利
- 功能清单：
  - 看自定义录入的套利监控项目。
- 数据库和数据来源：
  - `/api/monitors`
- 业务规则（核心）：
  - 页面必须显示公式、收购方股价、被收购方股价。
  - 股票腿理论对价 = 收购方股价 * 换股比例 * 安全系数 + 现金分派
  - 股票腿收益率 = (股票腿理论对价 - 目标现价) / 目标现价 * 100
  - 现金腿收益率 = (现金对价 - 目标现价) / 目标现价 * 100
  - 纯现金模式继续保留。
  - 默认返回结果要使用最新股价，不允许长期沿用旧价格。
  - 页面必须能稳定显示接口已返回的监控列表，不能因为单独渲染路径失效而整块空白或报错。
  - 主表必须支持分页，每页固定 50 条。
  - 页面必须支持新增监控项目。
  - 页面必须支持修改已有监控的参数并原地保存。
  - 补充参数区直接显示在每个监控项目下方，不再保留单独的 `详情` 标签或展开按钮。
  - 补充参数区文案必须使用真实业务口径，不能出现用户不认可的虚构标签。
- UI界面：
  - 展示监控名称、收购方/目标方、两边股价、公式说明、股票腿对价、现金腿对价、两种收益率。
  - 面板顶部提供新增 / 编辑表单，列表区展示已保存项目。
- 配置项说明：
  - 刷新频率：`config.yaml > data_fetch.plugins.custom_monitor.refresh_interval_ms`

## 11. 收购私有
- 功能清单：
  - 看收购、并购、私有化相关公告。
- 数据库和数据来源：
  - `/api/market/merger`
  - 并购报告继续沿用现有后端能力
- 业务规则（核心）：
  - 页面必须列出全量公告，不再只看当天。
  - 默认按公告时间倒序。
  - 当 `announcementDate` 或 `announcementTime` 属于当天时，该行高亮显示。
  - 主表要尽量展示现有可用字段，例如代码、公司、类型、关键词、公告标题、公告时间、现价、报价、报价溢价、链接。
  - 主表必须支持分页，每页固定 50 条。
- UI界面：
  - 以主表为主，不再保留重复摘要卡。
  - 今日公告要有明显高亮和角标。
- 配置项说明：
  - 刷新时间：`config.yaml > data_fetch.plugins.merger.refresh_time`
  - 回看天数：`config.yaml > data_fetch.plugins.merger.announcement_lookback_days`

- 数据库和数据来源：
  - 继续读取现有仪表盘接口。

  - 电脑端强调层次、字号可读性和表格利用率。

## 13. 验收标准
- 首页打开后必须能看到真实数据。
- 六个功能页必须能切换。
- 顶部状态信息是一行文字，不再是大状态卡。
- `股债打新` 是今日事项长表，并显示关键日期和申购上限等字段。
- `股债打新` 不再显示 `抽签日` 列，页面上的 `中签缴款日` 使用 `lotteryDate` 的日期值。
- `股债打新` 的 `今日中签缴款` 阶段与页面显示保持同口径，也按 `lotteryDate` 判定。
- `推送设置` 位于页面末尾，只保留 3 个时间框，且能保存并立即回显。
- `转债套利 / AH / AB` 主表都新增序号列。
- `转债套利 / AH / AB` 主表都支持点击表头排序。
- `转债套利 / AH / AB / 监控套利 / 分红提醒 / 收购私有` 主表都支持分页，每页 50 条。
- `监控套利` 支持新增与编辑已有监控，且补充参数直接显示在项目下方，不再依赖单独详情按钮。
- AH/AB 同时显示前三和倒数前三。
- AH/AB 主表显示价差、近三年百分位和压缩样本区间。
- `收购私有` 显示全量公告，今日公告行高亮明显。
- `转债套利` 顶部摘要区明显收紧，主表在首屏更早可见。
- 电脑端页面接近满屏显示，整体字号明显提升，手机端页面不压坏。
## 14. 云服务器公网访问
- 功能清单：
  - 项目必须部署在云服务器上，并保持长期在线，任何人拿到网址后都能直接打开首页。
- 数据库和数据来源：
  - 页面与接口继续读取当前项目已有数据库、运行时状态文件和 `/api/*` 数据源，不因为部署到云服务器而改变业务数据来源。
- 业务规则（核心）：
  - 对外访问目标是“固定网址长期可用”，不接受“只在本机能打开”或“重启后要手动重新跑命令”。
  - 最终对外入口优先使用 `80/443`，并优先提供 HTTPS。
  - 裸 `5000` 端口只允许作为调试或过渡访问方式，不能作为最终正式对外方案。
  - 服务进程必须由正式守护方式托管，例如 `systemd`、等价服务管理器或容器重启策略。
  - 服务器重启后，网页服务必须自动恢复。
  - 云防火墙、安全组、系统防火墙和反向代理配置必须互相一致，不能只放行其中一层。
  - 对外健康检查必须能访问，并能区分 `web`、`data_jobs`、`push_scheduler` 的状态。
- UI界面：
  - 公网访问时首页地址保持同一套页面，不新增第二套“外网版页面”。
  - 若启用域名，浏览器打开域名后直接进入现有仪表盘首页。
- 配置项说明：
  - 服务监听地址：`config.yaml > app.host`
  - 服务端口：`config.yaml > app.port`
  - 对外基础地址：`config.yaml > app.server_base_url`

## 15. 公网部署验收
- 外网设备在不连接服务器本机桌面的情况下，直接访问正式网址可以打开首页。
- 云服务器重启后，不需要人工登录执行命令，网页仍能恢复访问。
- 手动杀掉网页主进程后，守护机制能自动拉起服务。
- 正式网址访问优先为 HTTPS；如果暂未配置域名，则至少要证明公网 IP 可访问，并明确这只是过渡状态。
- `/api/health` 通过正式网址可访问，且 `web` 状态为 `ok`。

## 16. GitHub 自动更新部署
- 功能清单：
  - 仓库推送到 `main` 后，云服务器自动拉取最新代码并完成部署。
- 数据库和数据来源：
  - 自动部署只更新代码和依赖，不改变当前业务数据库、运行数据目录和 `/api/*` 数据来源。
- 业务规则（核心）：
  - 自动部署正式链路固定为：`GitHub Actions -> SSH -> 服务器更新脚本 -> 服务重启 -> 健康检查`
  - GitHub Actions 工作流不得在 YAML 内直接执行 `git fetch/reset`、`npm ci/install`、`systemctl restart` 等服务器业务步骤
  - 代码同步、依赖同步、服务重启、健康检查统一由 `tools/deploy/update_from_github.sh` 承担
  - 工作流日志必须明确标记至少 4 个阶段：`Validate secrets`、`Prepare SSH`、`Trigger remote script`、`Deployment done`
  - 正式自动部署目标分支固定为 `main`
  - 服务器项目目录默认是 `/home/ubuntu/Alpha monitor`
  - 自动部署完成后，若 `alpha-monitor` 服务存在，必须自动重启该服务
