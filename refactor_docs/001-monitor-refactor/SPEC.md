# Alpha Monitor 生效规格说明

状态：当前生效文档  
日期：2026-04-22  
用途：描述当前项目的正式目标状态、模块边界、页面行为、推送规则、配置合同与验收口径。  
说明：本文件只保留当前生效内容，不记录历史 phase 变更日志。

## 1. 文档定位

- `REQUIREMENTS.md` 用于说明产品需求与业务目的。
- `SPEC.md` 用于定义当前批准实施的正式规格。
- `plan.md` 只记录本次正在执行的一轮计划。

本文件要求：

- 面向人阅读
- 以当前生效状态为准
- 不堆叠历史变更日志
- 控制在 500 行以内

## 2. 系统目标

Alpha Monitor 的目标是：

- 用真实数据生成套利监控网页
- 提供定时推送与独立策略推送
- 交付为云端可访问服务，而不是仅本机演示

统一原则：

- 只使用真实数据、真实接口、真实文件结果
- 页面优先服务看盘与决策
- 推送必须与页面口径一致
- 配置必须集中在 `config.yaml`

## 3. 架构边界

### 3.1 职责分层

- `data_fetch/`：抓取、标准化、缓存真实数据
- `strategy/`：计算业务指标、过滤规则、生成候选结果
- `presentation/`：页面、接口整形、视图模型
- `notification/`：推送配置、推送文案、调度
- `shared/`：配置、路径、时间、运行态、公共工具

### 3.2 边界约束

- 展示层不做业务计算
- 策略层不直接发送推送
- 推送样式层不重新发明业务口径
- 运行态 JSON 必须独立持久化，不混写到无关状态文件

## 4. 页面总规则

### 4.1 模块范围

当前主页面包含：

1. `转债套利`
2. `AH溢价`
3. `AB溢价`
4. `LOF套利`
5. `监控套利`
6. `分红提醒`
7. `事件套利`
8. `可转债抢权配售`

### 4.2 通用表格规则

除非模块单独说明，主表统一遵守：

- 支持排序、搜索、分页
- 页面默认每页 50 条
- 首次加载优先展示最近一次成功的真实数据
- 如果先读到缓存，前端同会话内必须触发一次后台重校验
- 空值显示 `--`，不允许伪造

### 4.3 桌面端与移动端

- 手机端与桌面端使用同一套模块顺序、字段语义和表格结构
- 不再维护单独移动端布局或移动端专属精简字段
- 手机端允许通过横向滚动查看长表，但信息口径必须与桌面端一致

### 4.4 自动刷新

- 页面使用轻量状态接口判断是否需要刷新
- 不应每分钟整页强刷全部重数据
- 关键行情模块允许读缓存后后台刷新

## 5. 转债套利

### 5.1 页面目标

页面用于展示可转债与正股的套利相关字段，重点服务于：

- 主表看盘
- 顶部摘要
- 独立低溢价监控与推送

### 5.2 主表基础字段

主表保留以下字段族：

- 行情：转债价、正股价、涨跌幅
- 转股相关：转股价、转股价值、转股溢价率、双低
- 折价策略：转债市值比、折价ATR比、市场
- 债底与期权：纯债价值、纯债溢价、理论价格、理论溢价率、理论期权价值、隐含期权价值、期权折价率
- 其他：剩余规模、波动率、剩余期限、评级、上市日、转股起始日

主表退役字段：

- `ATR系数`
- `抛压系数`
- `加权折价率`

### 5.3 主表计算口径

正式口径如下：

- `转股价值 = stockPrice * 100 / convertPrice`
- `转股溢价率 = (price / convertValue - 1) * 100`
- `双低 = price + premiumRate`
- `纯债溢价率 = (price / pureBondValue - 1) * 100`
- `ATR百分比 = stockAtr20 / stockPrice * 100`
- `转债市值比 = remainingSizeYi / stockMarketValueYi`
- `折价ATR比 = abs(premiumRate) / stockAtr20Pct`

约束：

- `stockMarketValueYi` 必须来自真实数据链路
- 如果 `stockMarketValueYi` 缺失或非正值，`转债市值比` 返回空值
- 如果 `stockAtr20Pct` 缺失或非正值，`折价ATR比` 返回空值
- 不允许用静态映射、写死值或伪造结果补空

### 5.4 理论定价口径

理论定价固定为“债底 + 净看涨价差价值”：

- `optionQty = 100 / convertPrice`
- `bondFloorStrike = pureBondValue / optionQty`
- `longCallStrike = max(convertPrice, bondFloorStrike)`
- `shortCallStrike = redeemTriggerPrice`
- `理论期权价值 = call(longCallStrike) - call(shortCallStrike)`
- `理论价格 = pureBondValue + max(理论期权价值, 0)`
- `理论溢价率 = (price / theoreticalPrice - 1) * 100`
- `隐含期权价值 = price - pureBondValue`
- `期权折价率 = implicitOptionValue / theoreticalOptionValue - 1`

### 5.5 样本过滤

失效样本不得进入主表可见结果：

- 退市
- 到期
- 终态强赎
- 停牌后终止交易

活跃强赎规则：

- 可在主表中保留风险提示表达
- 顶部摘要与独立折价推送排除活跃强赎样本

### 5.6 顶部摘要

顶部固定三组：

- `双低前3`
- `理论溢价前3`
- `低溢价监控`

规则：

- `双低前3` 按 `doubleLow` 升序
- `理论溢价前3` 按 `theoreticalPremiumRate` 降序
- `低溢价监控` 来自独立监控池
- 摘要必须排除终态样本与活跃强赎样本

### 5.7 低溢价监控规则

买卖阈值保持不变：

- 买入：`premiumRate < buy_threshold` 且已进入转股期
- 卖出：`premiumRate > sell_threshold`

说明：

- `转债市值比` 与 `折价ATR比` 只用于展示、排序和上下文提示
- 本轮不引入新的买卖触发公式
- 运行态为空时，如样本已处于买入区，首次 bootstrap 不能静默吞掉真实买入事件

### 5.8 排序规则

低溢价监控池默认排序：

1. `premiumRate` 更低者优先
2. `discountAtrRatio` 更高者优先
3. `bondToStockMarketValueRatio` 更低者优先
4. 代码升序作最终兜底

## 6. AH 溢价

### 6.1 目标

展示 A/H 溢价排行与分位情况。

### 6.2 正式口径

- `H股人民币价 = H股价 * HKD/CNY`
- `AH溢价 = (H股人民币价 / A股价 - 1) * 100`
- `价差 = H股人民币价 - A股价`

### 6.3 页面规则

- 支持排序、分页、搜索
- 显示近三年分位与样本区间
- 主摘要中只作为 AH 小节输出，不再与 AB 合并成一个总标题

## 7. AB 溢价

### 7.1 目标

展示 A/B 溢价排行与分位情况。

### 7.2 正式口径

- `B股人民币价 = B股价 * 对应汇率`
- `AB溢价 = (B股人民币价 / A股价 - 1) * 100`
- `价差 = B股人民币价 - A股价`

### 7.3 页面规则

- 布局、排序、分页和 AH 页面保持一致
- 主摘要中只作为 AB 小节输出

## 8. LOF 套利

### 8.1 目标

展示 LOF / QDII 的套利数据与监控池。

### 8.2 正式口径

- `溢价率 = (现价 / IOPV - 1) * 100`
- 指数 LOF / QDII 亚洲按净值日期与指数涨跌、汇率修正 IOPV

### 8.3 页面规则

- 顶部只显示限购监控池与非限购监控池
- 无真实输入时保留行，但不伪造 IOPV 或溢价率

## 9. 自定义监控

### 9.1 目标

展示用户维护的换股 / 现金监控列表。

### 9.2 正式口径

- 股票腿理论对价 = 收购方股价 × 换股比例 × 安全系数 + 现金对价
- 股票腿收益率 = (股票腿理论对价 - 目标现价) / 目标现价 × 100
- 现金腿收益率 = (现金选择权价格 - 目标现价) / 目标现价 × 100

### 9.3 页面规则

- 按最大收益率排序
- 保留明细说明
- 支持页内新增、编辑、删除

## 10. 分红提醒

### 10.1 目标

提醒登记日、除权日等分红事件。

### 10.2 正式口径

- `股息率 = 每股分红 / 当前股价 × 100`

### 10.3 页面规则

- 优先提醒今日登记日
- 展示完整观察名单

## 11. 事件套利

### 11.1 目标

统一展示港股私有化、中概私有化、A 股并购等事件机会。

### 11.2 页面规则

- 统一承接多个事件来源
- 缺少某一子类时允许显示空态或缓存态
- 不伪造结果

## 12. 可转债抢权配售

### 12.1 目标

展示抢权配售候选，按当前三阶段表达：

- `申购阶段`
- `埋伏阶段`
- `等待阶段`

### 12.2 正式口径

- `发行比例 = 发行规模 / 总市值`
- `配售股数 = ceil(原始所需股数 × 0.6 / 100) × 100`
- `两融所需股数 = ceil(原始所需股数 × 0.6 / 50) × 50`
- `所需资金 = 配售股数 × 当前股价`
- `两融所需资金 = 两融所需股数 × 当前股价`

### 12.3 页面规则

- 只保留一张主表
- 不再显示旧监控池表达
- `monitorList` 兼容字段可保留为空，但不再驱动页面含义

## 13. 推送系统

### 13.1 推送链路

当前真实生效的推送链路：

1. 主摘要定时推送
2. 可转债折价独立推送
3. 可转债抢权配售独立推送
4. LOF 套利独立推送

统一特点：

- 通道固定为企业微信 Webhook
- 推送末尾可追加网页入口链接
- 服务端是唯一主动调度方
- 页面只负责读状态、改配置

### 13.2 主摘要定时推送

主摘要仍是一份报告，但只保留以下 5 段，顺序固定：

1. `今日打新`
2. `可转债机会`
3. `自定义监控`
4. `AH`
5. `AB`

格式规则：

- 每段标题必须加粗
- 每段之间必须留空行
- `分红提醒` 与 `事件套利` 不再进入主摘要

### 13.3 可转债折价独立推送

推送时段与阈值规则保持不变，但内容改为：

- `可转债代码名称`
- `溢价率`
- `转债市值比`
- `折价ATR比`
- `是否当前为强赎（强赎中/非强赎）`

说明：

- 活跃强赎样本仍按原规则排除出折价推送候选
- 因此可推送样本通常显示 `非强赎`
- 推送中不再展示 `ATR系数 / 抛压系数 / 加权折价率`

### 13.4 LOF 独立推送

- 交易日定时推送
- 新入池项目允许即时推送
- 使用独立运行态做去重

### 13.5 抢权配售独立推送

- 按模块独立配置运行
- 不与主摘要时间表混用

## 14. 配置与运行态合同

### 14.1 正式配置入口

所有长期配置必须进入 `config.yaml`，重点包括：

- 页面入口与表格 UI
- 自动刷新
- 各模块策略参数
- 通知默认时间
- 运行态文件映射

### 14.2 关键运行态文件

至少包含：

- `push_config.json`
- `push_runtime_state.json`
- `cb_discount_strategy_state.json`
- `lof_arbitrage_push_runtime.json`
- 其他模块自己的运行态文件

### 14.3 折价策略运行态

`cb_discount_strategy_state.json` 至少保存：

- `initializedDate`
- `lastBootstrapDate`
- `monitorMap`
- `signalStateMap`
- `monitorPushRecords`
- 最近一次买入、卖出、监控推送的尝试/成功/错误时间

约束：

- 折价策略运行态独立持久化
- 不能复用主摘要冷却状态
- 缺状态文件时必须可初始化
- 初始化后不能无限重复 bootstrap
- bootstrap 快照发现样本已在买入区时，必须保留可触发的真实信号结果

## 16. 验证规则

### 16.1 最小校验

实施后至少优先执行：

- `npm run check`
- `npm run check:boundaries`
- `python data_dispatch.py exchange-rate`
- `python data_dispatch.py ah`
- `python data_dispatch.py ab`
- 本轮相关 Node 自动化测试

### 16.2 本轮重点验证

必须验证以下场景：

- `转债市值比` 与 `折价ATR比` 使用真实输入计算
- 旧折价因子链不再被页面和推送引用
- 折价推送内容与页面字段一致
- 缺失运行态时，折价策略能初始化并保持后续状态穿越判断有效
- 主摘要只保留 5 段，且顺序正确
- 推送文案在有空值时显示真实空态，不伪造

### 16.3 部署完成标准

完成后必须满足：

- 云端页面可访问
- `/api/health` 正常
- 相关接口返回结构可用
- 本轮修改无明显空值失真与推送异常
