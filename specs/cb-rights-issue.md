# 转债抢权配售策略

## 1. 策略目标

在真实市场数据中识别转债抢权配售机会，形成独立功能链路。

首期目标：新增独立功能页、页面展示监控列表、通过固定来源真实数据计算配售所需资金/配售预期收益/预计收益率、对满足门槛项目做独立定期推送。

首期不做：异动推送、公告页解析、用户侧 URL 输入。固定来源：`https://www.jisilu.cn/web/data/cb/pre`。

## 2. 页面与产品结构

`转债抢权配售` 页面拆为三个子标签页：
- `申购阶段`：`inApplyStage = true`
- `埋伏阶段`：未进入申购阶段，且阶段属于 `上市委通过/同意注册/注册生效`，同时 `预期收益率 > 6%`
- `等待阶段`：除上述两组之外的全部项目

主表固定展示字段：正股代码、正股名称、方案进展、进展公告日、发行规模、总市值、发行比例、原始所需股数、配售股数、两融所需股数、转股价、波动率、单位期权价值、期权价值、所需资金、股权登记日、预期收益率、两融收益率、预期收益率去皮、两融收益率去皮。

页面不显示：转债代码、转债名称。

`股权登记日` 只在 `申购阶段` 显示，其余阶段不显示。三个子标签页均不显示 `年化收益率` 列。

## 3. 核心计算口径

### 3.1 发行比例

`发行比例 = 发行规模 / 正股总市值`

- `发行规模` 使用固定源中的真实 `amount / amountYi`
- `总市值` 必须直接调用真实股票实时接口获取

### 3.2 股数与资金

- `原始所需股数 = rawRequiredShares`
- `配售股数 = 原始所需股数`
- `两融所需股数 = ceil((原始所需股数 × 0.6) / 50) × 50`
- `所需资金 = 配售股数 × 当前股价`
- `两融所需资金 = 两融所需股数 × 当前股价`

### 3.3 收益率、去皮与年化

- `期权价值 = 单位期权价值 × 期权数量`
- `预期收益率 = 期权价值 / 所需资金`
- `两融收益率 = 期权价值 / 两融所需资金`

去皮收益率固定为额外释放资金的价值：
- `原始资金基线 = 原始所需股数 × 当前股价`
- `预期收益率去皮 = 预期收益率 × (原始资金基线 - 所需资金) / 所需资金`
- `两融收益率去皮 = 两融收益率 × (原始资金基线 - 两融所需资金) / 两融所需资金`

年化收益率：`年化收益率 = (1 + 两融收益率去皮)^(252 / 预计交易日数) - 1`

### 3.4 预计交易日数

- 若已给出 `applyDate`，优先按 `今天 -> applyDate` 的真实交易日数计算
- 若还未给出 `applyDate`：`上市委通过->申购日` 和 `同意注册/注册生效->申购日` 均使用真实历史样本交易日差中位数
- 若无真实样本，`预计交易日数` 和 `年化收益率` 都必须为空

### 3.5 单位期权价值

- 行权价：`max(URL 页面给出的转股价, 当前价)`
- 无风险利率：十年期国债收益率
- 波动率：正股 `60` 日历史波动率（基于后复权数据）
- 存续期：`6 年`
- `期权数量 = 1000 ÷ 行权价`

### 3.6 正股历史数据库

独立数据库，只用于计算 `60 日历史波动率`：
- 存储监控范围内正股的 `后复权` 历史行情
- 每天补齐最新一个交易日数据
- 随固定来源页面项目增减同步增减股票覆盖范围

## 4. 推送规则

独立推送只保留两组：
1. `申购阶段项目`
2. `预期收益率 > 6% 项目`（排除已在第一组出现的重复项目）

默认交易日推送时间：`08:00`、`14:30`。

推送正文必须包含：方案进展、进展公告日、发行规模、总市值、发行比例、两融所需股数、两融收益率、两融收益率去皮、年化收益率。

## 5. API

### `GET /api/market/cb-rights-issue`

返回：当天监控列表、更新时间、最近一次重建状态。

兼容处理：`data.monitorList` 保留但固定返回空数组，不再代表有效监控集合。

### `GET/POST /api/push/cb-rights-issue-config`

读写独立推送配置。

## 6. 运行态持久化

独立保存：当天监控列表、固定来源结构化快照、当日推送记录、最近一次重建时间/错误。

运行态必须与现有主推送、转债折价监控、监控套利运行态隔离。

## 7. 首期边界

不做：任意公告页解析、异动推送、人工候选池与正式池双层管理、跨天保留旧监控池、主观评分系统、假数据演示、用户手动粘贴 URL 解析入口。

做：独立页面、三阶段分组列表、固定来源结构化信息区、正式入池判断、每日 00:00 重建、独立定期推送、独立正股历史数据库。

## 8. 验证要求

- 首页新增 `转债抢权配售` 根 Tab
- 只有处于 `上市委通过/同意注册/已明确申购日` 的项目有资格入池
- 预计收益率 `<= 6%` 的项目不会进入正式监控池
- 深市、沪市配售股数按既定规则计算；沪市 `0.6` 修正规则生效
- `配售所需资金/配售预期收益/预计收益率` 三项结果闭环一致
- 正股 `60 日波动率` 来自独立历史数据库
- 推送配置独立保存，默认时间 `08:00`、`14:30`
- `打新/转债/AH/AB/监控/分红/事件套利` 现有行为不回退
