# IOPV 24h 价格优化方案

> 状态：方案阶段，未实现
> 作者：MiMo
> 日期：2026-05-30

## 1. 问题背景

当前 IOPV 计算使用 Yahoo 日频复权收盘价作为美股持仓的 `current_prices`：

| 时段 | 当前行为 | 问题 |
|------|---------|------|
| A股交易时段 (9:30-15:00 CST) | 用美股上一交易日收盘价 | 盘后新闻/财报未反映 |
| 美股盘后 (16:00-20:00 ET) | 同上 | 同上 |
| 美股盘前 (04:00-09:30 ET) | 同上 | 盘前大幅波动未反映 |

**核心矛盾**：A股交易时段（北京时间9:30-15:00）对应美东时间21:30-03:00（夏令时），正好覆盖美股盘后时段。当前只用收盘价，错过了盘后4小时的价格变动。

## 2. 目标

用 Yahoo 5m 盘前盘后价格替代日频收盘价，使 IOPV 在 A股交易时段反映美股盘后/盘前的最新价格。

**只影响 IOPV 计算，不影响回测、DB存储、其他模块。**

## 3. 数据验证结果

2026-05-30 实测 34 个标的，Yahoo Deno 代理 `includePrePost=true` + `interval=5m`：

### 3.1 覆盖率

| 类型 | 标的数 | 盘后有数据 | 5m可用 | 备注 |
|------|--------|-----------|--------|------|
| ETF (美股) | 20 | 20/20 (100%) | 20/20 | 全部可用 |
| 个股 (美股) | 10 | 10/10 (100%) | 10/10 | 全部可用 |
| 指数 (^NDX等) | 3 | 3/3 (100%) | 3/3 | 盘后=收盘，无差异 |
| 期货 (GC=F) | 1 | 1/1 (100%) | 1/1 | 盘后活跃 |

**结论：所有标的均可用，覆盖率 100%。**

### 3.2 盘后价格差异（vs 日频收盘）

| 差异范围 | 标的数 | 占比 | 代表标的 |
|----------|--------|------|---------|
| < 0.2% | 18 | 53% | GLD, QQQ, SOXX, GOOGL |
| 0.2% ~ 0.5% | 10 | 29% | KWEB, XOP, ARKK, PDD |
| 0.5% ~ 1.0% | 4 | 12% | INDA, BOTZ, GC=F |
| > 1.0% | 2 | 6% | IXC(+1.29%), OILK(-1.24%) |

**多数标的盘后差异 < 0.5%，对 IOPV 影响有限。少数流动性较差标的差异较大。**

### 3.3 特殊情况

| 情况 | 处理 |
|------|------|
| RYH 无 5m 数据 | fallback 到日频收盘价 |
| 指数 (^NDX, ^GSPC) | 盘后=收盘，无额外数据 |
| 非交易日 | 返回最近交易日收盘价 |

## 4. 方案设计

### 4.1 架构

```
source.py (数据获取层)
  ├── 港股持仓 → 腾讯行情（不变）
  ├── 美股持仓 → Yahoo 5m pre/post（新增）
  │     ├── 成功 → 用最新盘后价
  │     └── 失败/无数据 → fallback 到日频收盘价
  └── DB fallback（不变）

calc.py (计算层)
  └── calc_iopv()（不变，输入数据口径变了）
```

### 4.2 数据流

```
Yahoo Deno 代理
  /v8/finance/chart/{ticker}?range=5d&interval=5m&includePrePost=true
      │
      ▼
  返回 5 分钟 K 线（含盘前盘后）
      │
      ▼
  取最后一条非 None close 作为 current_price
      │
      ▼
  calc_iopv(current_prices, nav_date_prices, ...)
```

### 4.3 API 调用量变化

| 指标 | 当前（日频） | 优化后（5m） | 变化 |
|------|------------|------------|------|
| 每标的每次请求 | 1 次 (1d, 5d range) | 1 次 (5m, 5d range) | 不变 |
| 返回数据量 | ~5 条/标的 | ~900 条/标的 | ×180 |
| 响应时间 | ~1s | ~2s | ×2 |
| 总标的数 | 34 | 34 | 不变 |
| 刷新频率 | 5 分钟 | 5 分钟 | 不变 |

**API 调用次数不变，但每次返回数据量增大 180 倍。Deno 代理带宽压力增加。**

### 4.4 代码变更范围

| 文件 | 变更 |
|------|------|
| `data_fetch/lof_iopv/source.py` | 美股持仓 fetch 改用 5m pre/post，fallback 到日频 |
| `data_fetch/lof_iopv/yahoo_finance.py` | `fetch_history()` 支持 `include_pre_post` 参数 |
| `strategy/lof_iopv/calc.py` | **不改** |
| `strategy/lof_iopv/service.py` | **不改** |
| `data_fetch/lof_db/etf_updater.py` | **不改**（DB 仍存日频收盘价） |

## 5. IOPV 影响估算

以 161126（标普医疗保健，持仓 RYH）为例：

```
当前：IOPV = NAV × (1 + 0.946 × (-0.064%)) × fx = 1.8345
盘后 RYH 涨 0.5%：IOPV = NAV × (1 + 0.946 × 0.436%) × fx = 1.8428
差异：+0.45%（对应溢价率变化约 0.45%）
```

对高溢价标的（如 501225 溢价 42%），盘后价格变动对 IOPV 的影响被溢价率放大，但相对溢价率本身仍然很小。

## 6. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 盘后流动性差，价格噪声大 | IOPV 波动增加 | 可加平滑/中位数过滤 |
| Deno 代理带宽压力 | 响应变慢 | 5m 数据量大但 gzip 压缩后可控 |
| 部分标的无 5m 数据 | IOPV 缺失 | fallback 到日频收盘价 |
| 时区转换错误 | 价格取错 | 用 UTC 时间戳，不依赖本地时区 |
| 期货 24h 交易 | 盘后价格持续变动 | 期货本身就需要最新价，是优势 |
