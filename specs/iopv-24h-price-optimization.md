# IOPV 实时价格优化方案

> 状态：方案阶段，未实现
> 作者：MiMo
> 日期：2026-05-31（v2 重写）

## 1. 问题

当前 IOPV 用**日频收盘价**计算，A股交易时段（9:30-15:00 CST）对应的美股价格是上一交易日收盘价。
这意味着美股盘后4小时+盘前5.5小时的价格变动完全不反映在 IOPV 中。

**目标**：IOPV 使用尽可能新的价格，覆盖盘前/盘中/盘后/夜盘全时段。

## 2. 核心策略：ETF → 期货替代

ETF 是期货的衍生品，价格天然滞后于期货。直接用期货价格替代 ETF，可以：
- 获得接近 24h 的连续报价（期货交易时间 ~23h/天）
- 消除 ETF 跟踪误差
- 消除 ETF 盘后流动性差的问题

## 3. 数据获取方案

### 3.1 三层价格优先级

```
current_prices 获取顺序：
  1. Yahoo 5m 最新价（含盘前盘后，期货天然24h）
     ├── 成功 → 使用
     └── 失败 → ↓
  2. 腾讯实时行情（港股/A股持仓）
     ├── 成功 → 使用
     └── 失败 → ↓
  3. DB 历史收盘价（etf_prices/stock_prices）
```

### 3.2 Yahoo API 调用方式

```
GET /v8/finance/chart/{ticker}?range=5d&interval=5m&includePrePost=true

标的类型     interval    includePrePost    覆盖时间
期货(F)      5m          不需要(天然24h)    ~23h/天
ETF/个股      5m          true              盘前+盘中+盘后 ~16h
指数(^)       1d          不需要            仅盘中
```

**关键点**：期货不需要 `includePrePost` 参数，因为期货市场本身就是接近 24h 交易。

### 3.3 数据量与性能

| 指标 | 当前(日频) | 优化后(5m) | 期货(5m) |
|------|-----------|-----------|---------|
| 每标的数据量 | ~5条 | ~900条 | ~1069条 |
| 响应时间 | ~1s | ~2s | ~2s |
| API调用次数 | 不变 | 不变 | 不变 |
| 总标的数 | 34 | 34 | 34 |

**带宽优化**：只取最后一条有效价格即可，不需要全部数据。可考虑新增 `fetch_latest_price_5m()` 函数，只返回最新一条。

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 期货与ETF价格基差 | IOPV偏离实际净值 | 多数情况下基差<0.5%，可接受 |
| 期货合约换月 | 价格跳变 | Yahoo主力合约自动处理 |
| 盘后流动性差(ETF) | 价格噪声 | 期货替代后此问题消失 |
| 5m数据量增大 | 带宽压力 | 只取最新一条，不取全量 |
| RYH等无期货标的 | 仍用ETF日频 | ETF盘前盘后数据兜底 |

## 8. 不做的事

- 不改回测逻辑（回测用日频收盘价）
- 不改 DB 存储（etf_prices 仍存日频收盘价）
- 不改港股/A股价格来源
- 不改 calc.py 计算公式（只改输入数据源）
