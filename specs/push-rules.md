# 推送规则

**Created**: 2026-04-30
**Last Updated**: 2026-04-30
**Status**: active

## 1. 设计原则

- 规则先于配置：先定义推送内容/时间/条件，再谈可配置项
- 推送内容必须来自真实数据，推送时强制刷新数据源
- 状态机驱动实时推送，只在状态变化时触发
- 定时推送按固定时间表，不依赖事件触发

## 2. 推送链路总览

| 链路 | 类型 | 推送时间 | 通道 |
|------|------|----------|------|
| 主摘要定时推送 | 定时 | 08:00, 20:18 | 企业微信 |
| 转债套利折价策略提醒 | 实时（状态机） | 交易时段 09:30~14:50 | 企业微信 |
| 转债抢权配售独立推送 | 定时 | 08:00, 14:30 | 企业微信 |
| LOF 套利独立推送 | 定时+即时 | 13:30, 14:00, 14:30 + 新入池 | 企业微信 |
| 并购报告推送 | 定时 | 00:00 | 企业微信 |

## 3. 主摘要定时推送

### 3.1 推送内容

聚合以下模块的摘要数据，拼装为一份 Markdown 报告：

| 段落 | 数据源 | TOP N |
|------|--------|-------|
| 今日打新 | `subscription` | 全部 |
| 转债套利机会 | `cb_arb` | 双低 TOP 3、理论溢价 TOP 3、回售 TOP 3、涨停 TOP 3、转股套利 TOP 3、低于面值 TOP 3 |
| 自定义监控 | `custom_monitor` | 全部 |
| AH 溢价 | `ah_premium` | 最高/最低各 TOP 3 |
| AB 溢价 | `ab_premium` | 最高/最低各 TOP 3 |

分红提醒与事件套利**不纳入**主摘要。

### 3.2 推送时间

固定时间点：`08:00`、`20:18`

### 3.3 触发条件

- 定时触发，不依赖事件
- `tradingDaysOnly` 为 `false` 时非交易日也推送
- 若某模块数据为空，该段落显示"暂无数据"但不跳过整份推送

### 3.4 推送格式

Markdown，各段标题加粗，段之间留空行。

## 4. 转债套利折价策略提醒

### 4.1 推送内容

3 类独立提醒：

| 提醒类型 | 触发条件 | 推送字段 |
|----------|----------|----------|
| 买入提醒 | 标的首次进入买入区 | 代码、名称、价格、溢价率、正股价格、正股涨跌幅、转股价值、强赎状态 |
| 卖出提醒 | 监控名单中的标的进入卖出区 | 同上 |
| 监控名单推送 | 定时推送当前监控名单 | 同上 |

### 4.2 买入区判定

```
premiumRate < -2%   // 买入阈值
&& hasPassedConvertStart   // 已过转股日
```

### 4.3 卖出区判定

```
premiumRate > -0.5%   // 卖出阈值
&& 该标的在监控名单中
```

### 4.4 推送时间

交易时段内 `09:30 ~ 14:50`，按 10 分钟一档检查。

买入/卖出提醒：状态变化时即时推送。
监控名单：按时间表推送，已推送的时段不重复。

### 4.5 状态机规则

- 首次启动（`initializedDate` 为空）：仅初始化监控名单，不推送
- 后续启动：只在状态变化时触发买入/卖出推送
- 标的进入买入区 → 加入监控名单
- 标的进入卖出区 → 从监控名单移除
- 监控名单中的标的若数据缺失（退市/过期）→ 自动移除

### 4.6 数据源刷新

推送前强制刷新 `cbArb` 数据（`force: true`），确保溢价率为实时值。

### 4.7 推送格式示例

```
# 转债套利低溢价买入提醒
> 2026-04-30 10:30:00
> 触发条件：转股溢价率低于买入阈值

- 123045 某转债 | 价格 95.50 | 溢价率 -3.50% | 正股 12.50(+1.20%) | 转股价值 98.20 | 非强赎
```

## 5. 转债抢权配售独立推送

### 5.1 推送内容

抢权配售模块正式入池项目列表。

### 5.2 推送时间

`08:00`、`14:30`

### 5.3 触发条件

- 交易日才推送
- 监控名单为空则跳过，不记成功
- 不并入主摘要

### 5.4 推送格式

Markdown，独立推送。

## 6. LOF 套利独立推送

### 6.1 推送内容

LOF 限购池 / 非限购池项目。

### 6.2 推送时间

定时：`13:30`、`14:00`、`14:30`
即时：新入池时触发

### 6.3 触发条件

- 定时推送按时间表执行
- 即时推送只在新入池时触发
- `seenEntryMap` 去重，避免同一标的重复即时推送

### 6.4 推送格式

Markdown，独立推送。

## 7. 并购报告推送

### 7.1 推送内容

DeepSeek AI 生成的单家公司并购简报。

### 7.2 推送时间

每日 `00:00` 抓取后触发。

### 7.3 触发条件

- 基于当日公告数据生成
- 逐家公司推送

### 7.4 推送格式

Markdown，报告长度限制 `500` 字符。

## 8. 通用规则

### 8.1 数据真实性

- 所有推送数据必须来自真实接口
- 推送前强制刷新对应数据源
- 任一字段抓不到时显示 `--`，不伪造

### 8.2 通道与格式

- 通道：企业微信 Webhook
- 格式：Markdown
- 末尾可追加网页入口链接（若配置了 `PUSH_HTML_URL`）

### 8.3 本地开发禁用

`PUBLIC_BASE_URL` 为 `127.0.0.1 / localhost / 0.0.0.0` 时，推送调度自动禁用。

## 9. API 与页面

| 端点 | 说明 |
|------|------|
| `GET /api/push/rules` | 返回推送规则定义（JSON） |
| `GET /push-rules` | 推送规则展示页面（HTML） |
| `POST /api/push/wecom` | 手工触发主摘要推送 |
| `POST /api/push/wecom/event-alerts` | 手工触发折价策略推送 |

## 10. 验收标准

- **AC-001**: 折价策略推送前强制刷新数据源，溢价率与页面实时值一致
- **AC-002**: 状态机只在变化时触发推送，同一标的不重复推送
- **AC-003**: 主摘要各段落格式统一，空数据段显示"暂无数据"
- **AC-004**: `/push-rules` 页面展示所有推送链路的规则
