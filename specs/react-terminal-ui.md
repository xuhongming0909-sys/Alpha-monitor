# React 金融终端 UI 规格

**Created**: 2026-04-30
**Last Updated**: 2026-05-06
**Status**: active

## 1. 范围

本规格定义 React + Vite 金融终端 UI 的并行重做范围、接口消费清单和验收标准。

本轮不改变业务计算口径、数据抓取规则、推送规则和现有接口含义。

## 2. 路由与托管

- 默认首页 `/` 托管 React + Vite 构建产物，静态产物目录为 `ui/dist`
- 旧 HTML 看板通过 `/legacy` 保留，继续使用原 `presentation.dashboard_entry` 指向的模板文件
- 根项目必须提供 `npm run ui:build` 与 `npm run ui:check`

## 3. 接口消费清单

React UI 必须消费以下现有真实接口：

- `GET /api/health`
- `GET /api/dashboard/ui-config`
- `GET /api/dashboard/resource-status`
- `GET /api/market/convertible-bond-arbitrage`
- `GET /api/market/ah`
- `GET /api/market/ab`
- `GET /api/market/lof-arbitrage`
- `GET /api/market/cb-rights-issue`
- `GET /api/monitors`
- `GET /api/market/subscriptions`

不允许使用假数据、演示数据或静态占位数据冒充接口结果。

## 4. Tab 结构（7个顶级标签）

```
概览 | 转债套利 | AH溢价 | AB溢价 | LOF套利 | 打新申购 | 自定义
```

### 4.1 概览

**布局（从上到下，分块摘要流）：**

1. **系统摘要块**
   - 数据刷新状态
   - 关键资源状态
   - 核心数量摘要：转债样本、AH 数量、AB 数量、LOF 数量、自定义数量、今日申购/中签/上市

2. **打新摘要块**
   - 今日申购、今日中签缴款、今日上市三类摘要行
   - 下钻入口指向“打新申购”标签
   - 数据来源：`/api/market/subscriptions`

3. **转债摘要块**
   - 主页样本数
   - 折价套利数量
   - 理论折价套利数量
   - 小额刚兑数量
   - 抢权配售阶段摘要

4. **AH/AB 摘要块**
   - AH 顶部机会摘要
   - AB 顶部机会摘要

5. **LOF 摘要块**
   - 高溢价/低溢价摘要
   - 顶部机会摘要

6. **自定义摘要块**
   - 监控条目数
   - 最高收益摘要
   - 收购方/目标方关键标的摘要

**概览限制：**
- 概览只展示摘要流，不再嵌入桌面完整宽表格
- 概览不展示分红提醒、事件套利、推送设置入口或摘要
- 摘要块顺序固定，块内使用简洁表格或极简列表展示

### 4.2 转债套利（5个子Tab）

```
主页 | 折价套利 | 理论折价套利 | 小额刚兑 | 抢权配售
```

**主页**：完整转债表格，22列：
转债、代码、转债价、涨跌、正股、正股价、正股成交(万)、转股价、转股价值、溢价率、转债市值比、折价ATR、市场、双低、纯债溢价、理论溢价、理论期权、隐含期权、波动率、剩余期限、规模、状态

**折价套利**：`premiumRate < 0` 的转债（同主页列）

**理论折价套利**：`price < theoreticalPrice` 的转债（同主页列）

**小额刚兑**：17列完整小额刚兑表格

**抢权配售**：沪市/深市子Tab × 申购/埋伏/等待三阶段，含所有字段

### 4.3 AH溢价（13列）

序号、A股名称、A股代码、H股名称、H股代码、A股价格、H股价格、H股人民币价、价差、溢价率、近三年分位、样本数、样本区间

### 4.4 AB溢价（13列）

序号、A股名称、A股代码、B股名称、B股代码、A股价格、B股价格、B股人民币价、价差、溢价率、近三年分位、样本数、样本区间

### 4.5 LOF套利（15列）

名称、代码、现价、涨幅、成交额(万)、净值、净值日期、相关指数、申购状态、申购费、赎回状态、赎回费、托管费、IOPV、溢价率

### 4.6 打新/申购

React 标签文案固定为：**打新申购**

名称、代码、类型、申购日、中签缴款日、上市日、申购上限、发行价、转股价

### 4.7 自定义监控

React 标签文案固定为：**自定义**

页面必须拆为 3 个区块，按顺序展示：

1. **收购方表**
   - 监控名称、收购方名称、收购方代码、收购方市场、收购方原币种、收购方原始价格、收购方人民币价格、换股比例、安全系数

2. **目标方表**
   - 监控名称、目标方名称、目标方代码、目标方市场、目标方原币种、目标方原始价格、目标方人民币价格、备注

3. **收益表**
   - 监控名称、股票腿收益率、现金腿收益率、最优收益率、现金对价、现金选择权、收益排序位次

三表共用同一批真实监控数据，不额外定义新接口。

## 5. 导航移除范围

- React 顶层导航移除：分红提醒、事件套利、推送设置
- React 概览移除：分红提醒摘要、事件套利摘要、推送设置摘要
- 上述模块仍可保留后端能力、旧看板能力和独立规格，但不属于本轮 React 导航范围

## 6. 样式规范

- 终端风格：深色背景（#050608），高密度数据展示
- 主展示形态：所有设备统一 React 模块表格，视觉风格尽量还原旧网页端的干净清爽布局
- 表格：模块头 + 表头 + 数据行，窄屏允许横向滚动
- 字段：等宽数字、轻装饰、少层级，数字右对齐
- 字体：等宽字体用于代码/数字
- 响应式：不得恢复 1920px/1800px 桌面锁死宽表；横向滚动只用于单行字段完整展示

## 7. 验收标准

- `npm run ui:build` 通过
- 所有测试通过：`node tests/ui_*.test.js`
- 7个顶级标签全部可访问
- 转债套利5个子Tab全部可访问且有数据
- 抢权配售正常显示数据（stockMarketValueYi依赖后端填充）
- 概览页按分块摘要流展示，不出现旧版综合大表
- React 导航中不存在分红提醒、事件套利、推送设置
- “自定义”页面按收购方表、目标方表、收益表三段展示
- 标的列表为简洁模块表格，窄屏可横向滚动查看完整字段
