# React 金融终端 UI 规格

**Created**: 2026-04-30
**Last Updated**: 2026-04-30
**Status**: active

## 1. 范围

本规格定义 React + Vite 金融终端 UI 的并行重做范围、接口消费清单和验收标准。

本轮不改变业务计算口径、数据抓取规则、推送规则和现有接口含义。

## 2. 路由与托管

- 默认首页 `/` 托管 React + Vite 构建产物，静态产物目录为 `ui/dist`
- 旧 HTML 看板通过 `/legacy` 保留，继续使用原 `presentation.dashboard_entry` 指向的模板文件
- 根项目必须提供 `npm run ui:build` 与 `npm run ui:check`

## 3. 接口消费清单

React UI 必须消费以下现有真实接口：

- `GET /api/health`
- `GET /api/dashboard/ui-config`
- `GET /api/dashboard/resource-status`
- `GET /api/market/convertible-bond-arbitrage`
- `GET /api/market/ah`
- `GET /api/market/ab`
- `GET /api/market/lof-arbitrage`
- `GET /api/market/cb-rights-issue`
- `GET /api/market/event-arbitrage`
- `GET /api/monitors`
- `GET /api/market/subscriptions`
- `GET /api/dividend?action=portfolio`
- `GET /api/push/config`

不允许使用假数据、演示数据或静态占位数据冒充接口结果。

## 4. Tab 结构（9个顶级Tab）

```
概览 | 转债套利 | AH溢价 | AB溢价 | LOF套利 | 打新/申购 | 自定义监控 | 分红提醒 | 事件套利
```

### 4.1 概览

**布局（从上到下）：**

1. **打新置顶区**（高亮区块，优先展示）
   - 今日申购/中签/上市卡片（带数字徽章）
   - 完整打新表格：当前阶段（今日申购/今日中签缴款/今日上市高亮标签）、类型（新股/债券）、名称/代码、申购日、中签缴款日、上市日、申购上限、发行价/转股价
   - 数据来源：`/api/market/subscriptions`

2. **机会总览区**（全策略混排）
   - 筛选器：`[全部|转债|LOF|AH|AB|事件套利]` 策略类型
   - 表格列：标的、代码、策略、核心指标、数值、约束/风险、报价
   - 数据来源：各策略接口机会数据，按核心指标排序取前18条
   - 排序规则：按核心指标绝对值降序

3. **转债主页表格**（完整22列，同下4.2主页规格）

**指标卡片行：** 机会总数、转债样本（含小额刚兑）、AH/AB 数量、LOF 数量、自定义监控数量、今日申购/中签/上市

### 4.2 转债套利（5个子Tab）

```
主页 | 折价套利 | 理论折价套利 | 小额刚兑 | 抢权配售
```

**主页**：完整转债表格，22列：
转债、代码、转债价、涨跌、正股、正股价、正股成交(万)、转股价、转股价值、溢价率、转债市值比、折价ATR、市场、双低、纯债溢价、理论溢价、理论期权、隐含期权、波动率、剩余期限、规模、状态

**折价套利**：`premiumRate < 0` 的转债（同主页列）

**理论折价套利**：`price < theoreticalPrice` 的转债（同主页列）

**小额刚兑**：17列完整小额刚兑表格

**抢权配售**：沪市/深市子Tab × 申购/埋伏/等待三阶段，含所有字段

### 4.3 AH溢价（13列）

序号、A股名称、A股代码、H股名称、H股代码、A股价格、H股价格、H股人民币价、价差、溢价率、近三年分位、样本数、样本区间

### 4.4 AB溢价（13列）

序号、A股名称、A股代码、B股名称、B股代码、A股价格、B股价格、B股人民币价、价差、溢价率、近三年分位、样本数、样本区间

### 4.5 LOF套利（15列）

名称、代码、现价、涨幅、成交额(万)、净值、净值日期、相关指数、申购状态、申购费、赎回状态、赎回费、托管费、IOPV、溢价率

### 4.6 打新/申购

名称、代码、类型、申购日、中签缴款日、上市日、申购上限、发行价、转股价

### 4.7 自定义监控

监控名称、收购方→目标方、目标现价、股票腿收益率、现金腿收益率、最优收益率、备注

### 4.8 分红提醒

代码、名称、登记日、除权日、派息日、每股分红、股息率、当前价

### 4.9 事件套利（4个子Tab）

```
A股套利 | 港股套利 | 中概私有 | 最新公告
```

**A股套利**：代码、名称、现价、涨跌幅、安全边际价、安全边际折价、现金选择权价格、现金选择权折价、币种、事件类型、官方公告

**港股套利**：代码、名称、现价、涨跌幅、市值、私有化价格、套利空间、私有化进程、要约方、要约方持股、注册地、收购方式、是否可反套、是否可卖空、核心公告

**中概私有**：代码、名称、现价、涨跌幅、市值、私有化价格、套利空间、进程、要约方、收购方式、详情链接

**最新公告**：公告时间、公司/代码、交易类型、报价、目标现价、报价溢价、链接

## 5. 推送设置（独立Tab）

推送设置作为独立顶级Tab，不在其他Tab底部重复显示。

Tab名称：**推送设置**

显示内容：
- 推送总开关状态
- 推送时间配置
- 各模块开关状态（转债、AH/AB、LOF、打新、监控、分红、事件套利）
- Webhook配置状态
- 调度器运行状态
- 上次推送时间

## 6. 样式规范

- 终端风格：深色背景（#050608），高密度数据展示
- 表格：所有单元格内容居左对齐，数字列右对齐
- 字体：等宽字体用于代码/数字
- 响应式：最小宽度1920px，表格支持水平滚动

## 7. 验收标准

- `npm run ui:build` 通过
- 所有测试通过：`node tests/ui_*.test.js`
- 9个顶级Tab全部可访问
- 转债套利5个子Tab全部可访问且有数据
- 事件套利4个子Tab全部可访问
- 抢权配售正常显示数据（stockMarketValueYi依赖后端填充）
- 推送设置作为独立Tab存在
- 表格数字列右对齐居中显示
