# 可转债折价策略

## 1. 策略目标

本策略用于在真实盘中数据里识别“可转债价格低于转股价值、且折价质量较高”的机会，并把提醒拆成三段式链路：

- 买入提醒：发现新的高质量折价机会时即时提醒
- 卖出提醒：已进入监控名单的标的折价明显收敛时即时提醒
- 持仓监控推送：对监控名单做交易时段内的持续跟踪

本策略直接替代当前仅按 `转股溢价率 < -3%` 触发的旧可转债折价异动规则，但不影响 AH、AB、打新、分红、自定义监控等其他推送模块。

## 2. 适用范围与准入条件

### 2.1 标的范围

候选池为全市场可转债，但买入判断只对以下标的生效：

- 存续中
- 可交易
- 未退市、未到期、未完成强赎
- 已进入转股期

首期继续复用当前仓库里可转债服务的清洗逻辑：

- `strategy/convertible_bond/service.js` 已具备失效标的过滤
- 已覆盖退市、到期、停牌后终止、完成强赎等典型失效场景

### 2.2 不作为硬门槛的信息

以下信息首期只展示，不参与买入硬过滤：

- 正股是否为融资融券标的

原因：

- 两融属性能提升可操作性判断，但不是绝对必要条件
- 先避免因为数据缺口或硬过滤而错杀真实机会

## 3. 核心定义与计算口径

### 3.1 折价率

正式定义：

`折价率 = -转股溢价率

### 3.2 加权折价率

核心排序指标：

`加权折价率 = 折价率 × ATR系数 × 抛压系数 × 板块系数` #加权折价率用%表示

说明：

- 加权折价率用于排序和监控优先级
- 买入硬阈值仍以原始折价率判断，不直接用加权折价率代替买点

### 3.3 ATR 百分比口径

仓库中已有字段 `stockAtr20`，其含义是正股近 20 日 ATR 的绝对价格值。策略正式口径改为百分比：

`ATR百分比 = stockAtr20 / 正股现价 × 100`

进一步定义：

`ATR比值 = 折价率 / ATR百分比`

解释：

- 这样折价率与 ATR 都是百分比口径，量纲一致
- 折价率越大、ATR 百分比越小，说明当前折价相对正股波动越“厚”

### 3.4 ATR 系数

ATR 系数根据 `ATR比值` 按锚点做分段线性插值，并将结果限制在 `[0, 1]`：

- `0 -> 0`
- `0.25 -> 0.5`
- `0.5 -> 0.8`
- `1 -> 1`

执行规则：

- `ATR比值 <= 0`，ATR 系数记为 `0`
- `0 < ATR比值 < 1`，按相邻锚点做线性插值
- `ATR比值 >= 1`，ATR 系数记为 `1`

直观理解：

- 折价厚度不足以覆盖正股短期波动时，系数明显打折
- 折价厚度超过 ATR 百分比后，认为波动风险被较充分覆盖

### 3.5 抛压比值与抛压系数

定义：

`抛压比值 = 剩余规模 / 正股近20日平均成交额`

仓库已有字段：

- `remainingSizeYi`
- `stockAvgTurnoverAmount20Yi`

抛压系数按以下锚点分段线性插值：

- `0.5 -> 1`
- `1 -> 0.8`
- `5 -> 0.5`
- `20 -> 0`

执行规则：

- `抛压比值 <= 0.5`，抛压系数记为 `1`
- `0.5 < 抛压比值 < 20`，按相邻锚点做线性插值
- `抛压比值 >= 20`，抛压系数记为 `0`

说明：

- 原始讨论里使用“无穷大 -> 0”
- 为了让公式可以直接实施，首期将“无穷大”落成有限上限 `20`
- 该阈值后续可以配置化，但正式文档先按 `20` 固定

直观理解：

- 剩余规模相对正股成交额越小，转股后的潜在抛压越容易被市场承接
- 比值很高时，即使当前有折价，也要明显打折

### 3.6 板块系数

固定值如下：

- 科创板：`1`
- 创业板：`0.85`
- 主板：`0.8`

首期板块识别规则：

- 正股代码前缀为 `688` 时记为科创板
- 正股代码前缀为 `300` 或 `301` 时记为创业板
- 其余 A 股默认记为主板

说明：

- 当前仓库尚无统一的正股板块字段
- 首期先按代码规则派生，后续如补入权威板块字段，再由真实字段替换

## 4. 买卖与监控规则

### 4.1 买入提醒

买入提醒同时满足以下条件时触发：

- 折价率 `> 2%`
- 已进入转股期
- 标的当前有效，未失效
- 本轮扫描前不在“买入区”，本轮扫描后进入“买入区”

这里的“进入买入区”采用阈值穿越逻辑，而不是条件满足就反复提醒。

### 4.2 卖出提醒

卖出提醒只对已在监控名单中的标的生效，触发条件为：

- 折价率 `< 0.5%`
- 本轮扫描前不在“卖出区”，本轮扫描后进入“卖出区”

卖出提醒触发后，标的自动移出监控名单。

### 4.3 监控名单

监控名单由系统自动维护：

- 命中买入提醒后自动加入
- 命中卖出提醒后自动移出

监控名单排序规则：

- 按 `加权折价率` 从高到低排序
- 同分时优先折价率更高者
- 再同分时优先 ATR 系数更高者

### 4.4 监控推送频率

监控名单定时推送规则固定为：

- 仅在沪深交易时段运行
- 上午按 `09:30，09:40, 09:50, 10:00 ... 11:30`
- 下午按 `13:00，13:10, 13:20, 13:30 ... 14:50`
- `14:50` 必须推送一次，即使当日没有新增买卖信号也要推

无监控标的时不发送空消息。

## 5. 推送内容设计

### 5.1 买入提醒

标题建议：

`可转债折价买入提醒`

每条展示以下字段：

- 转债名称 / 代码
- 正股名称 / 代码
- 正股是否融资融券标的
- 折价率
- ATR 系数
- 折价率 / ATR
- 抛压系数
- 剩余规模 / 正股近 20 日平均成交额
- 板块
- 加权折价率

建议排版：

```text
- 海优转债 123xxx | 海优新材 688xxx | 折价 2.35% | 加权折价 1.64%
  ATR系数 0.82 | 折价/ATR 0.56 | 抛压系数 1.00 | 抛压比 0.42 | 科创板 | 两融 是
```

### 5.2 卖出提醒

标题建议：

`可转债折价卖出提醒`

每条字段可比买入提醒略简，但必须保留：

- 转债名称 / 代码
- 正股名称 / 代码
- 当前折价率
- 当前加权折价率
- 触发原因：折价率低于 `0.5%`

### 5.3 监控名单推送

标题建议：

`可转债折价监控`

内容规则：

- 仅推当前监控名单
- 按加权折价率排序
- 包含转债名称 / 代码，正股名称 / 代码，正股是否融资融券标的， 折价率，加权折价率，板块

## 7. 后续实现职责拆分

### 7.1 strategy/convertible_bond

负责：

- 候选池清洗与过滤
- 折价率、ATR 百分比、ATR 系数、抛压系数、板块系数、加权折价率计算
- 买入/卖出区间判定
- 买卖信号生成

不负责：

- 企业微信文案拼装
- 冷却判断
- 页面展示

### 7.2 notification/alerts

负责：

- 买入提醒即时推送
- 卖出提醒即时推送

要求：

- 退役旧的 `convertPremiumLt` 可转债异动规则
- 不再使用当前“折价阈值 + 冷却分钟数”的旧模型
- 改成“基于状态穿越的即时提醒”

### 7.3 notification/summary 或独立监控推送模块

负责：

- 监控名单的定时推送
- 交易时段内每 10 分钟推送
- 14:50 强制推送

### 7.4 runtime_data

为降低耦合，策略状态应独立持久化，不与当前推送冷却状态混写。

建议新增独立状态文件：

- `runtime_data/shared/cb_discount_strategy_state.json`

至少持久化以下内容：

- 当前监控名单
- 每只债最近一次是否处于买入区
- 每只债最近一次是否处于卖出区
- 最近一次买入提醒时间
- 最近一次卖出提醒时间
- 最近一次监控推送时间

### 7.5 config.yaml

后续实施时需要正式进入配置的参数：

- 买入阈值，默认 `2`

- 监控强制推送时点，默认 `14:50`

## 8. 与当前系统的替代关系

当前系统里已有一条旧规则：

- 可转债 `premiumRate < convertPremiumLt`

该规则问题：

- 只看负溢价绝对值
- 没有纳入正股波动、剩余规模、承接能力、板块差异
- 只有单次异动视角，没有完整的“买入-监控-卖出”链路

本策略落地后：

- 旧的可转债折价异动规则退役
- 其他模块异动推送保持不变
- 可转债异动逻辑统一由本策略接管

## 9. 验证要求

后续实施时至少验证以下场景：

- 转股溢价率为负时，折价率是否正确转成正值口径
- `stockAtr20` 转为 ATR 百分比后，ATR 系数是否按锚点正确插值
- 剩余规模和 20 日成交额齐全时，抛压系数是否正确计算
- 已进入转股期且折价率从 `1.8%` 升到 `2.1%` 时，是否只触发一次买入提醒
- 已在监控名单中的标的，折价率从 `0.8%` 降到 `0.4%` 时，是否只触发一次卖出提醒并自动移出
- 监控名单是否按加权折价率排序
- 交易时段内是否按 10 分钟节奏推送
- `14:50` 是否固定推送
- 无监控名单时是否不发空消息

## 10. 当前默认假设

本文件先采用以下默认假设，后续如实盘观察需要，再单独修订：

- 折价率为=-转股溢价率
- ATR 统一采用 `近20日ATR / 正股现价`
- 两融属性仅展示，不做买入硬门槛
- 板块首期按正股代码前缀映射
- 抛压系数的“无穷大 -> 0”在实施上落成 `20 -> 0`
- 监控名单自动进入、自动退出，不依赖人工维护

## 11. 2026-03-30 最新口径更新

- `ATR系数` 当前正式口径为：
  - `ATR百分比 = stockAtr20 / stockPrice × 100`
  - `溢价绝对值 = abs(-premiumRate)`
  - `atrCoefficient = 溢价绝对值 / ATR百分比`
- `抛压系数` 已简化为直接等于：
  - `20日均成交额 / 剩余规模`
- `市场系数` 保持原板块映射方案不变。
- `理论期权价值` 当前多头看涨行权价正式口径为：
  - `optionQty = 100 / 转股价`
  - `债底折算行权价 = 纯债价值 / optionQty`
  - `longCallStrike = max(转股价, 债底折算行权价)`
  - `shortCallStrike = 强赎价`
  - `理论期权价值 = call(longCallStrike) - call(shortCallStrike)`
- `加权折价率` 现行公式为：
  - `weightedDiscountRate = (-premiumRate) × atrCoefficient × sellPressureCoefficient × boardCoefficient`
- 旧版 ATR / 抛压锚点插值口径已退役，不再参与实时页面与接口计算。
