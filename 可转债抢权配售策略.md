# 可转债抢权配售策略

## 1. 策略目标

本策略用于在真实市场数据中识别“可转债抢权配售”机会，并形成一条独立于现有转债套利、AH/AB、监控套利、分红提醒、事件套利之外的新功能链路。

首期目标固定为四件事：

1. 新增一个独立功能页 `可转债抢权配售`
2. 页面顶部展示 `监控列表`
3. 通过固定来源的真实数据计算配售所需资金、配售预期收益和预计收益率
4. 对满足正式门槛的项目做独立定期推送

本策略首期不做异动推送，不接入任意公告页，也不开放用户侧 URL 输入。

首期固定只使用你提供的这一个来源链接作为入口：

- `https://www.jisilu.cn/web/data/cb/pre`

## 2. 页面与产品结构

`可转债抢权配售` 页面自上而下固定为三个区域：

1. `监控列表`
2. `固定来源结构化信息区`
3. `页面注释`

### 2.1 监控列表

监控列表只展示“当天有效、自动入池”的正式项目，默认按 `预计收益率` 从高到低排序。

每条至少展示：

- 转债名称 / 代码
- 正股名称 / 代码
- 当前阶段
- 申购日
- 股权登记日
- 配售 10 张实际所需股数
- 配售所需资金
- 配售预期收益
- 预计收益率
- 原始 URL

### 2.2 固定来源结构化信息区

页面不再单独设置 `URL 解析区`，而是直接读取固定来源页面并展示结构化摘要。

固定来源：

- `https://www.jisilu.cn/web/data/cb/pre`

系统职责：

- 固定抓取该页面真实内容
- 提取结构化字段
- 补充正股价格、波动率、国债收益率等真实数据
- 计算配售相关指标
- 若满足正式门槛，自动加入当天监控列表

结构化信息区默认展示摘要，不直接堆整页原文。

必须保留以下信息：

- 原始来源链接
- 页面标题
- 转债基础信息
- 正股基础信息
- 发行进展信息
- 申购日 / 股权登记日等关键时间
- 与配售测算直接相关的原始字段
- 页面原始摘要文本或关键原文片段

### 2.3 页面注释

页面末尾必须补充：

- 数据来源
- 计算公式
- 策略说明

用于解释本页的真实数据来源、口径与边界，不改变计算结果。

## 3. 正式监控池规则

### 3.1 正式入池条件

正式监控池只收录同时满足以下条件的项目：

1. 发行进展处于关键可执行阶段
2. 预计收益率 `> 6%`

### 3.2 关键发行阶段

首期正式口径固定为以下三类：

- 上市委通过
- 同意注册 / 注册生效
- 已明确申购日的申购阶段

其中“申购阶段”的口径固定为：

- 只要已经明确申购日，即可进入正式监控
- 不再额外要求“提前 3 天”或“提前 5 天”的窗口规则

### 3.3 每日重建规则

监控列表是“当天有效池”，每天 `00:00` 执行全量重建：

1. 清空前一日监控列表
2. 重新抓取并重算当天可用项目
3. 仅把当天仍满足阶段与收益门槛的项目重新入池

URL 手动解析得到的项目如果满足门槛，也会进入当天监控列表；跨天后是否继续保留，由次日重建结果决定。

## 4. 核心计算口径

### 4.1 主公式

配售价值计算口径固定为：

- `配售所需资金 = 配售10张实际所需股数 × 当前股价`
- `配售预期收益 = 单位期权价值 × 期权数量`
- `预计收益率 = 配售预期收益 ÷ 配售所需资金`

### 4.2 配售 10 张实际所需股数

首期按图片规则正式落地：

#### 深市

- 先计算配售 10 张原始所需股数
- 再按 `100 股` 向上取整

#### 沪市

- 先计算配售 10 张原始所需股数
- 再乘以 `0.6`
- 再按 `100 股` 向上取整
说明：

- `0.6` 修正规则在首版中视为正式业务口径
- 不作为待验证项，不在实现时再临时变更

### 4.3 单位期权价值

单位期权价值按图片方案固定为估算口径：

- 行权价口径：`行权价 = max(URL 页面给出的转股价, 当前价)`
- 无风险利率：十年期国债收益率
- 波动率：正股 `60` 日历史波动率
- 存续期：`6 年`

其中 URL 页面给出的 `转股价` 在本策略中视为页面提供的 `20 日均值代理值`，因此行权价不再依赖本地均价库，而是直接与当前价比较取大值。

其中 `60 日历史波动率` 的计算依赖正股历史数据，不能只靠当天页面快照直接得到。

因此本功能仍然保留独立的正股历史数据库，但该数据库只用于计算 `60 日历史波动率`，并且使用 `后复权` 数据。

### 4.4 正股历史数据库规则

为了计算 `60 日历史波动率`，本功能仍然维护一套独立的正股历史数据库，但该库只用于波动率计算。

数据库职责固定为：

- 存储本功能监控范围内正股的 `后复权` 历史行情
- 为 `60 日历史波动率` 提供真实计算基础
- 每天补齐最新一个交易日数据
- 不参与行权价判定
- 随固定来源页面项目增减而同步增减股票覆盖范围

维护规则：

1. 当固定来源页面出现新的可转债抢权配售项目时：
   - 若其正股尚未进入该数据库，则新增该股票的历史数据追踪
2. 当固定来源页面中某个项目长期不再存在或已明确失效时：
   - 允许将对应正股从该数据库的有效跟踪列表中移除
3. 数据库中的每只正股数据都要每天更新到最新交易日
4. `60 日历史波动率` 必须基于该数据库中的真实 `后复权` 历史数据计算，不得使用静态填充值

说明：

- 这是 `cb_rights_issue` 功能自己的独立数据库，不与现有其他功能数据库混用为单一入口
- 该数据库只用于波动率，不用于行权价或 20 日均值判定

### 4.5 期权数量

固定公式：

- `期权数量 = 1000 ÷ 行权价`

### 4.6 风险边界

首期不引入额外风险折扣项：

- 不把持股波动损失单独折减进收益率公式
- 不引入人工主观打分
- 不做静态演示值

页面注释必须明确说明：

- 这是“配售期权价值估算口径”
- 不代表实际收益保证

## 5. 数据来源与职责拆分

### 5.1 数据来源

首期真实数据来源至少包含：

- 固定来源页面 `https://www.jisilu.cn/web/data/cb/pre`
- 正股实时价格
- 十年期国债收益率
- 正股独立历史数据库
- 基于独立历史数据库计算的正股 60 日历史波动率

### 5.2 分层职责

新功能必须使用独立链路 `cb_rights_issue`，不继续堆入现有 `subscription` 或 `convertible_bond` 页面代码。

#### data_fetch/cb_rights_issue

只负责：

- 抓取集思录预案 / 详情页
- 维护本功能独立正股历史数据库的股票覆盖范围
- 更新正股最新历史行情
- 标准化页面字段
- 补充正股行情和必要市场数据

不负责：

- 配售收益计算
- 入池判断
- 页面渲染
- 推送文案

#### strategy/cb_rights_issue

只负责：

- 阶段判定
- 配售股数计算
- 配售所需资金计算
- 单位期权价值计算
- 配售预期收益计算
- 预计收益率计算
- 正式入池判定
- 监控列表排序

#### presentation

只负责：

- 新页面展示
- API 整形
- 固定来源结构化信息展示
- 监控列表展示
- 推送时间设置展示

#### notification

只负责：

- 独立定期推送
- 推送格式
- 推送调度

### 5.3 错误处理规则

若遇到以下情况：

- 页面抓取失败
- 字段缺失
- 页面结构变化
- 关键市场数据缺失
- 正股历史数据库数据不足，无法计算 60 日波动率

系统必须：

- 明确返回失败原因
- 不用假数据补字段
- 不自动入池

## 6. API 与运行态规划

### 6.1 新增接口

首期最小必要接口为：

#### `GET /api/market/cb-rights-issue`

返回：

- 当天监控列表
- 更新时间
- 最近一次重建状态

#### `GET /api/push/cb-rights-issue-config`

返回：

- 独立推送配置

#### `POST /api/push/cb-rights-issue-config`

保存：

- 页面上的独立推送时间设置

### 6.2 运行态持久化

运行态需要独立保存：

- 当天监控列表
- 固定来源结构化快照
- 当日推送记录
- 最近一次重建时间
- 最近一次重建错误

说明：

- 该运行态必须与现有主推送、转债折价监控、监控套利运行态隔离
- 不得复用现有无关 JSON 状态文件

## 7. 推送规则

### 7.1 推送定位

本功能推送属于 `独立模块的定期推送`：

- 不并入现有 AH / AB 主汇总推送
- 不接入异动推送
- 不和转债折价监控共用推送时间

### 7.2 默认时间

默认交易日推送时间固定为：

- `08:00`
- `14:30`

### 7.3 页面设置

新页面需要提供独立时间选择框，支持读取和保存该模块自己的推送时间。

### 7.4 推送内容

推送内容基于“当天监控列表”生成，至少包含：

- 转债名称 / 代码
- 正股名称 / 代码
- 当前阶段
- 股权登记日
- 配售 10 张实际所需股数
- 配售所需资金
- 配售预期收益
- 预计收益率
- 原始 URL

## 8. 首期边界

本轮明确不做：

- 任意公告页解析
- 异动推送
- 人工候选池与正式池双层管理
- 跨天保留旧监控池
- 主观评分系统
- 假数据演示
- 用户手动粘贴 URL 解析入口

本轮明确做：

- 独立页面
- 监控列表
- 固定来源结构化信息区
- 正式入池判断
- 每日 00:00 重建
- 独立定期推送
- 独立正股历史数据库

## 9. 验证要求

正式实施时至少验证以下场景：

### 9.1 页面与交互

- 首页新增 `可转债抢权配售` 根 Tab

- 只有处于 `上市委通过 / 同意注册 / 已明确申购日` 的项目有资格入池
- 预计收益率 `<= 6%` 的项目不会进入正式监控池

### 9.3 公式与字段

- 深市、沪市 `配售10张实际所需股数` 按既定规则计算
- 沪市 `0.6` 修正规则生效

- 独立历史数据库每天补齐最新一个交易日数据

### 9.4 推送

- 该功能推送配置独立保存
- 默认时间为交易日 `08:00`、`14:30`
- 推送内容只包含该模块监控列表，不混入 AH / AB 内容

### 9.5 回归

- `打新 / 转债 / AH / AB / 监控 / 分红 / 事件套利` 现有行为不回退
- 固定来源抓取失败时不污染监控列表

本节覆盖本策略中此前与页面结构、发行比例、两融股数和推送范围不一致的旧口径。

### 11.1 页面结构改版

`可转债抢权配售` 页面正式改为只保留一张主表：

主表固定展示：

- 正股代码
- 正股名称
- 方案进展
- 进展公告日
- 发行规模
- 总市值
- 发行比例
- 原始所需股数
- 配售股数
- 两融所需股数
- 转股价
- 波动率
- 单位期权价值
- 期权价值
- 所需资金
- 股权登记日
- 预期收益率
- 两融收益率
- 预期收益率去皮
- 两融收益率去皮
- 年化收益率

页面不再显示：

- 转债代码
- 转债名称

### 11.2 发行比例口径

`发行比例` 的正式口径改为：

- `发行比例 = 发行规模 / 正股总市值`

其中：

- `发行规模` 继续使用固定源中的真实 `cb_amount`
- `总市值` 必须直接调用真实股票实时接口获取
- 不允许继续用配售股数、原始所需股数或历史比例反推总市值

### 11.3 股数与资金口径

本轮正式收口为：

- `原始所需股数 = rawRequiredShares`
- `配售股数 = 原始所需股数`
- `两融所需股数 = ceil((原始所需股数 × 0.6) / 50) × 50`
- `所需资金 = 配售股数 × 当前股价`
- `两融所需资金 = 两融所需股数 × 当前股价`

旧的“沪市先乘 0.6 后按 100 股取整作为主展示股数”的口径在本策略中正式废止。

### 11.4 收益率、去皮与年化口径

本轮正式口径为：

- `期权价值 = 单位期权价值 × 期权数量`
- `预期收益率 = 期权价值 / 所需资金`
- `两融收益率 = 期权价值 / 两融所需资金`

去皮收益率固定为“额外释放资金的价值”：

- `原始资金基线 = 原始所需股数 × 当前股价`
- `预期收益率去皮 = 预期收益率 × (原始资金基线 - 所需资金) / 所需资金`
- `两融收益率去皮 = 两融收益率 × (原始资金基线 - 两融所需资金) / 两融所需资金`

年化收益率固定为：

- 基数只取 `两融收益率去皮`
- `年化收益率 = (1 + 两融收益率去皮)^(252 / 预计交易日数) - 1`

### 11.5 预计交易日数

`预计交易日数` 的正式规则为：

- 若当前项目已给出未来 `applyDate`，优先按 `今天 -> applyDate` 的真实交易日数计算
- 若还未给出 `applyDate`：
  - `上市委通过 -> 申购日` 使用真实历史样本交易日差中位数
  - `同意注册 / 注册生效 -> 申购日` 使用真实历史样本交易日差中位数
- 若无真实样本，则 `预计交易日数` 和 `年化收益率` 都必须为空

### 11.6 列表置顶规则

页面不再用顶部卡片表达重点项目，改为列表内部置顶：

1. `进入申购阶段`
2. `预期收益率 > 6%`
3. 其他项目

其中“进入申购阶段”以 `applyDate` 有真实值为主，若阶段语义已明确进入申购，也必须与页面日期显示保持一致。

### 11.7 推送收口

独立推送只保留两组：

1. `申购阶段项目`
2. `预期收益率 > 6% 项目`

第二组必须排除已在第一组出现的重复项目。

推送正文必须包含：

- 方案进展
- 进展公告日
- 发行规模
- 总市值
- 发行比例
- 两融所需股数
- 两融收益率
- 两融收益率去皮
- 年化收益率

### 12.1 三个子标签页

页面正式拆为三个子标签页：

- `申购阶段`
- `埋伏阶段`
- `等待阶段`

分组规则固定为：

- `申购阶段`：`inApplyStage = true`
- `埋伏阶段`：未进入申购阶段，且阶段属于 `上市委通过 / 同意注册 / 注册生效`，同时 `预期收益率 > 6%`
- `等待阶段`：除上述两组之外的全部项目

### 12.2 取消原置顶表达

页面不再通过以下方式表达置顶：

- 正股名称下方的阶段徽标
- 正股名称下方的收益率徽标
- 行背景高亮

`正股名称` 单元格正式改为只显示正股名称文本。

### 12.3 字段差异

三个子标签页允许按阶段显示不同字段；本轮至少固定为：

- `股权登记日` 只在 `申购阶段` 显示
- `埋伏阶段` 与 `等待阶段` 不显示 `股权登记日`

### 12.4 年化收益率标签

本轮不新增任何 `年化收益率` 页面标签：

- 不在名称列下方显示
- 不在子标签页头部显示

如果页面继续保留 `年化收益率` 数据列，其字段口径与前文保持一致，但不再包装成额外标签。

## 13. 2026-04-17 发行规模纠偏与页面年化列下线

### 13.1 发行规模纠偏

抢权配售里的 `发行规模` 改回真实源字段：

- `issueScaleYi = amount / amountYi`

不再继续使用：

- `cb_amount / cbAmountYi`

也就是说，像 `金杨精密 / 金杨转债` 这类样本，页面与接口展示的 `发行规模` 应该回到真实的 `9.8` 亿级别，而不是把 `21.49` 当作发行规模。

### 13.2 页面年化列下线

抢权配售页面下的三个子标签页都不再显示 `年化收益率` 列：

- `申购阶段`
- `埋伏阶段`
- `等待阶段`

如仍需保留年化值，可保留在接口字段中，但页面不再直接展示。
